## Key Interactions to Verify

1. **Wake phrase activation** — user says "hey agent <command>" → HUD slides down, command executes.
2. **Ambient rejection** — user speaks without wake phrase → HUD stays hidden, no agent invocation, no Gemma call.
3. **Word-boundary rejection** — "hey agents of change" must NOT trigger the agent.
4. **Voice turn latency** — utterance end → first step_start event visible in HUD < 2.5s.
5. **Mode switch** — flipping Local-only ↔ Hybrid mid-session. Current turn completes in its starting mode; next turn uses new mode.
6. **Step status in HUD** — each step shows running (●) → success (✓) or failure (✗); retries increment a counter.
7. **Auto-hide** — 3s after `AGENT_DONE` the HUD fades out.
8. **Approval on write-actions** — draft email → HUD shows "Send?" → user must say "yes" before email actually sends.

## Edge Cases

- Wake phrase during a long background-speech utterance ("...and then I told him, hey agent, can you...")
- Overlapping speech (TV + user)
- Utterance with noise bleed-through
- Gemma returns malformed `<start_function_call>` token sequence → must escalate (hybrid) or fail-loud (local)
- MCP subprocess crashes mid-tool-call → router respawns + retries once, then reports to HUD
- Network drops during Gemini fallback → Local-only resolution or user-visible error
- Playwright selector disappears between DOM snapshot and click → Browser Use retries
- Cactus model OOM → reinit, show HUD error, don't crash
- Qt main thread starvation: agent posts 100 events in 50ms → HUD must not drop events
- User speaks DURING agent execution → ignored (no barge-in in v1); log for post-hackathon
- Hotkey not applicable (replaced by wake phrase)
- Empty transcript after silence → discard, don't route to planner

## Critical Paths (end-to-end)

1. **Full demo scenario (Hybrid):** "hey agent, check my labs and message Dr. Kim that I'd like a telehealth follow-up, then put a hold for Tuesday 2pm." Portal (Browser Use) + Gmail MCP + Calendar MCP, 3 tools, HUD shows 3 step groups.
2. **Local-only scenario:** "hey agent, read my latest email." Gmail MCP only, no Gemini calls (verify via HTTP mock that proxies 127.0.0.1 to assert no outbound Gemini request).
3. **Recovery:** inject a Playwright timeout during step 1 of the full demo → Browser Use retries → test passes.
4. **Approval:** "hey agent send an email to Alice saying hi" → HUD shows draft + "say yes to send" → speak "no" → email NOT sent, HUD shows cancelled.

## LLM/Prompt Test Requirements

- Eval suite: `tests/evals/test_tool_call_accuracy.py` with ≥30 voice-transcript → expected-tool-call pairs. Baseline: ≥85% exact match on Gemma local.
- Eval suite: wake-phrase false-positive rate on the Librispeech test-clean subset (random sentences) < 0.5%.
- Regression: when the prompt template in `cactus_chat_model.py` changes, both eval suites must be re-run and the baseline re-checked.
