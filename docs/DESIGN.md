## Problem Statement

There is a class of computer users who cannot reliably operate a physical keyboard and mouse — adults with ALS, late-stage cancer, severe arthritis, post-stroke motor impairment, Parkinson's tremor, or post-surgical recovery — but who can still speak and still need to do regular computer tasks: reading and replying to email, scheduling appointments, reviewing medical records on patient portals, browsing the web, managing calendars.

Today they have two broken options. Option one is OS-bundled accessibility tools (macOS Voice Control, Windows Voice Access, Dragon NaturallySpeaking) — command-and-control dictation. You say "click Submit" and they click Submit. They don't plan, don't reason, cannot handle "book my next oncology telehealth appointment on Tuesday afternoon and decline the overlapping meeting." Option two is modern agentic computer-use AI — Anthropic Claude Computer Use, OpenAI Operator, Browser Use, OpenAdapt. These plan and reason, but work by screenshotting the screen and shipping those screenshots to a cloud LLM. For a patient viewing an oncology portal, a clinician viewing Epic, or anyone bound by HIPAA, attorney-client privilege, or financial compliance — cloud round-trip is a non-starter.

The accessible tool can't plan. The agentic tool can't stay local. This product closes that gap.

## Demand Evidence

Weak-to-medium. Cloud-dependent computer use is demonstrably real (Operator launched, Claude Computer Use is GA, Browser Use has 50K+ GitHub stars). Adjacent accessibility demand is real (Dragon is a $300M+ business). The specific gap — local agentic computer use with voice — is unfilled; that is both the opportunity and the risk ("unfilled" could mean "too early" or "not painful enough").

No user interviews yet. No paying customer. No operating prototype. Team-scale conviction only. **The Assignment (bottom) addresses this with a deadline and an owner.**

## Status Quo

- Motor-impaired adult on macOS: **macOS Voice Control** (free, OS-bundled, command-level only — cannot chain multi-step tasks).
- On Windows: **Voice Access / Dragon** (same limitation; Dragon is $500+/seat).
- Clinician wanting a HIPAA-safe agent: **no good option** — types manually or uses dictation (Dragon Medical, Nuance). Cannot use Operator or CCU for HIPAA reasons.
- Lawyer: same as clinician.
- Privacy-sensitive consumer: manual computer use. Not a product they know they can replace.

**The real competitor in most cases is "typing manually, slowly, sometimes with family help."**

## Target User & Narrowest Wedge

**Hero user for the 90-second demo:** an adult with significant motor impairment operating a MacBook at home or in a care facility. Demo task is office-worker-style knowledge work: browsing, email, calendar, medical-portal messaging — voice-driven, data local.

**Narrowest demo slice (Hybrid mode):** a single contiguous voice-driven task touching 2–3 MCP tools end-to-end. Canonical scenario: *"Check my latest lab results on the patient portal, then send my oncologist a message saying I'd like to schedule a telehealth follow-up, then put a tentative calendar hold for Tuesday afternoon."* Exercises browser automation (portal), Gmail MCP (message), Calendar MCP (hold), with voice I/O throughout.

**Local-only mode demo:** a shorter, single-tool scripted task (e.g., "read my latest unread email from Dr. Kim"). This is a pitch asset, not a capability claim about matching Hybrid-mode depth.

**Beyond the demo:** company positions as "office workflows with local-first voice AI," with medical records as the foregrounded vertical. Adjacent verticals (legal, finance, government) are future.

## Premises (revised)

1. **Hero user** is a motor-impaired adult who can speak. Voice primary; text fallback.
2. **Privacy moat** — local execution is the differentiator vs. Operator and Claude Computer Use. Incumbents cannot trivially copy this.
3. **Technical feasibility, mode-differentiated:**
   - **Hybrid mode:** `functiongemma-270m` + Gemini fallback + MCP-native tool plane is sufficient for the 2–3-tool chained demo scenario. Gemini absorbs the "hard reasoning" (multi-step planning, ambiguous requests, response synthesis). Gemma's job in Hybrid mode is narrower: classify intent, map to a tool, fill slot values from the transcript.
   - **Local-only mode:** `functiongemma-270m` handles single-tool well-defined invocations via function-calling (that's the model's strength). It does NOT handle multi-step planning reliably. Local-only demo is one-shot scripted ("read X," "send Y") — short, clear, reliable.
4. **Architecture borrowing** — MCP-native: every capability is an MCP server call. Off-the-shelf servers for Gmail, Calendar, Browser. Custom servers only if none exists.
5. **Voice-primary interaction:** Whisper STT → planner → MCP action → macOS `say` TTS. Voice is the interface. Push-to-talk hotkey activates a turn.
6. **Dual mode:** Local-only vs Hybrid user-facing switch. Local-only = zero cloud, narrow capability, HIPAA/DoD-story. Hybrid = Gemini fallback for hard reasoning, consumer default.
7. **MCP-first with no bolted-on Accessibility-API code path in hackathon.** Any app lacking an MCP server is out of demo scope. Accessibility-API fallback is post-hackathon, NOT code we write this weekend.
8. **Scope discipline:** general architecture in code, ONE polished demo flow in video. Cuts declared upfront, not reactively.
9. **Target platform: macOS only for v1** (2026-04-18 decision). iOS/Windows/Linux are separate products, not ports. Captured in TODOS.md.

### Architecture

```
  USER VOICE (push-to-talk hotkey, Cmd+Shift+Space)
      │
      ▼
  Cactus Whisper-small (STT, on-device)
      │
      ▼
  Mode switch: [Local-only] vs [Hybrid]   ── CLI flag for hackathon; UI toggle post-hackathon
      │
      ▼
  Cactus Gemma functiongemma-270m-it (planner + tool-caller)
      │
      │   Hybrid mode only — escalation triggers:
      │     (a) Gemma returns a tool call to 'escalate'
      │     (b) 1 failed tool-call retry
      │     (c) user-visible "not sure — ask smarter model?" prompt
      │          (default ON in Hybrid, OFF in Local-only)
      │
      └──► Gemini 2.5 Flash (hard reasoning: multi-step planning, response synthesis,
                             ambiguity resolution)
              │
              ▼
        [back into Gemma with a reduced structured prompt]
      │
      ▼
  Capability router (implemented as a pre-registered MCP server manifest):
      planner outputs {tool: "<server>.<method>", args: {...}}
      router dispatches via the MCP Python SDK to the matching registered server
      │
      ├── browser-use-mcp-server      (web tasks, incl. mocked medical portal HTML)
      ├── gmail-mcp-server             (off-the-shelf; OAuth pre-configured on dev Mac)
      └── google-calendar-mcp-server  (off-the-shelf; same OAuth scope)
      │
      ▼
  Action result (JSON) + optional summary text
      │
      ▼
  Response synthesizer:
      Hybrid mode: Gemini 2.5 Flash writes the natural-language reply (higher quality)
      Local-only:  Gemma produces a short templated reply from the action result
      │
      ▼
  macOS `say` command → speaker
```

### Interaction & approval model

- **Activation:** push-to-talk hotkey (Cmd+Shift+Space). Press-and-hold to speak. Release to end turn. No wake-word in hackathon. No barge-in.
- **Approval policy:** **always confirm write-actions** (send email, create calendar event, submit form). **Never confirm read-actions** (read email, read calendar, load page, read lab). Agent voices a one-sentence summary then waits for "yes / no / edit" before executing writes.
- **Error handling:** on tool-call failure, agent retries once with identical args. On second failure, in Hybrid mode it escalates to Gemini with the error payload for a recovery plan. In Local-only mode it reports the error verbally ("I couldn't load the portal — please check your connection") and ends the turn. No silent failures.

### Latency budget (target per voice turn)

| Stage                  | Target     | Hard cap |
|------------------------|-----------:|---------:|
| STT (Whisper-small)    |   ≤500 ms  |  1000 ms |
| Planner (Gemma TTFT)   |   ≤400 ms  |   800 ms |
| Gemma decode (≤64 tok) |   ≤600 ms  |  1200 ms |
| Tool call (MCP)        |   varies   |  4000 ms |
| Response synth         |   ≤500 ms  |  1500 ms |
| TTS (`say`)            |   ≤300 ms  |   600 ms |
| **End-to-end turn**    | **~2.5 s** |  **<5 s** |

Hour-4 smoke test must measure end-to-end turn latency against these numbers.

### System prompt & tool-schema design

- **Planner system prompt** (Gemma): ~150 tokens. States: role (tool-calling assistant), available tools (dynamic list, inserted from MCP server manifest, ≤6 tools in context at once), output format (Gemma's native `<start_function_call>...<end_function_call>` tokens — already in the tokenizer), one few-shot example of a correct call.
- **Response-synth prompt** (Gemma, Local-only mode only): ~80 tokens. States: role (spoken reply writer), input (user utterance + tool result JSON), output format (1–2 short sentences, ≤30 words, no markdown).
- **KV cache discipline:** reset between voice turns (`cactus_reset`). Do not accumulate conversation history across turns in hackathon scope — it complicates the 270M model's reliability. Each turn is stateless from the model's perspective; the MCP server layer holds any persistent state (draft messages, etc.).

### Event contract (single source)

```python
# events.py
from dataclasses import dataclass
from enum import Enum
from typing import Literal

class EventType(str, Enum):
    WAKE_DETECTED = "wake_detected"
    STT_PARTIAL   = "stt_partial"
    STT_FINAL     = "stt_final"
    AGENT_START   = "agent_start"
    STEP_START    = "step_start"
    STEP_DONE     = "step_done"
    AGENT_DONE    = "agent_done"
    AGENT_ERROR   = "agent_error"
    MODE_CHANGE   = "mode_change"

@dataclass(frozen=True)
class AgentEvent:
    event_id: int
    type: EventType
    timestamp: float
    task: str | None = None
    step_index: int | None = None
    summary: str | None = None   # ≤120 chars
    success: bool | None = None
    error: str | None = None
    mode: Literal["local","hybrid"] | None = None
```

### Failure modes (new codepaths — see test plan for gaps)

- Continuous Whisper model hang → 5 s timeout + restart
- Gemma malformed function_call → retry once, escalate (hybrid) / error (local)
- MCP subprocess crash → respawn + retry, max 2
- Gemini HTTPS timeout/429 → `google-genai` built-in retries; fail to "cloud unreachable" event after 10 s
- Qt main thread blocked on paint → use QPropertyAnimation, never I/O on main thread — enforced by code review
- Wake-phrase false positive ("hey agents of change") → word-boundary regex `\b(hey\s+agent|okay\s+agent|agent,)\b`
- Cactus Gemma OOM under memory pressure → destroy + reinit; HUD "restart needed" event
- Browser Use Playwright selector miss → Browser Use retries 3× default, then raises → `ToolError` to HUD
- Vision tool: screen capture permission denied → onboarding flow in `main.py` prompts for TCC grant
- Vision tool: pyautogui blocked by macOS Accessibility permission → same TCC flow
